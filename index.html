Efficient Multiplier Architectures for CNN-Based Image Processor

Designing High-Performance Processing Elements for Efficient CNN Accelerators.

⚠️ Problem Statement

Convolutional Neural Networks (CNNs) are highly effective for image processing, but they are incredibly computationally expensive. Over 90% of a CNN's workload is dominated by Multiply-Accumulate (MAC) operations. In hardware implementation, traditional multipliers consume excessive power, occupy large silicon area, and suffer from high critical path delays.

💡 Our Solution

Efficient Multiplier Architectures for CNN-Based Image Processors. To overcome the MAC bottleneck, we designed an optimized hardware multiplier architecture utilizing Radix-4 Booth encoding and high-speed compressor trees. This enables highly efficient, real-time AI inference for modern image processing applications.

Architecture Benefits & Efficiency

1. Memory Tiling

Instead of fetching overlapping pixels from off-chip DRAM constantly, tiling caches local blocks in SRAM. This reuses data, drastically cutting memory bandwidth bottlenecks.

2. Radix-4 Booth Encoding

By grouping bits during the multiply phase, Radix-4 reduces the number of generated partial products by exactly 50% compared to standard multipliers.

3. 4:2 Compressor Tree

Instead of chaining traditional 3:2 Full Adders, 4:2 compressors condense four inputs into two outputs in parallel. This yields a much shorter critical path delay.
